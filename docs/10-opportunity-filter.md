# 10. OPPORTUNITY FILTER

Three questions. Reject anything small.

1. **Can it scale to $1M+?**
2. **Is it leverage-driven?**
3. **Does it build long-term wealth?**

## Modulor — tested against the filter

| Path | Scale to $1M+? | Leverage-driven? | Long-term wealth? | Verdict |
|------|----------------|------------------|-------------------|---------|
| **Hardware only** — ship boxes to teams | Soft pass. $50M ceiling (500 programs × $100k) | **No.** Every sale = install, train, support. Linear. | Only via acquisition. Fragile. | **Reject** |
| **Hardware + SaaS** — machine + per-athlete subscription | Pass. 50k pro/college/clinic athletes × $50/mo = $30M ARR | Partial. SaaS scales; hardware doesn't. | Recurring rev + data moat. | **Keep as wedge** |
| **Data licensing** — injury-risk model to insurers/leagues | **Strong pass.** Liberty Mutual writes $1B+ sports comp. 1% take = $10M/yr. | **Yes.** One model serves every customer. | Data compounds. Moat deepens monthly. | **Pursue** |
| **Protocol marketplace** — PTs publish, athletes subscribe | **Strong pass.** Winners-take-most. App-Store economics. | **Yes.** Others build content. You take the cut. | Platform = enterprise value multiple 10x SaaS. | **Pursue** |
| **Acquisition stack** — build to be bought | Pass if exit is $50M+ | Yes — build for the buyer's gap, not the market's | One liquidity event, then done | **Pursue in parallel** |
| **Services / custom installs** | Fail | No | No | **Reject** |
| **Content / courses / merch** | Fail | No | No | **Reject** |

## The honest read

**Current framing (hardware company) fails the filter.** $50M ceiling, linear labor, exit-dependent.

**Reframed (data + model company with hardware as the acquisition wedge) passes all three.** 
- Scale: $500M+ outcome realistic
- Leverage: software + data serve every customer at zero marginal cost
- Wealth: recurring revenue, data moat, strategic acquirer appetite

## What this means for the next move

| Do | Don't |
|----|-------|
| Build the data layer day one — every session logged, labeled, licensable | Sell machines as the product |
| Price the machine near cost to seed the data flywheel | Chase hardware margin |
| Structure every team deal with data-rights in perpetuity | Sign away data for pilot discount |
| Lease, don't sell — recurring revenue from day one | One-time invoices |
| Build the marketplace API into V1 | Treat software as bolt-on |
| Pitch strategics (Hyperice, Therabody, Nike, Arctos, RedBird) | Pitch generic VCs |

## The one-line test

Before any task, ask: **"Does this move us toward $500M or toward $5M?"**

If the answer is $5M, don't do it.
